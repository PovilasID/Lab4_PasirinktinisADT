<h1>Ketvirtas darbas</h1>
	
	<h3>Laboratorinis darbas: Duomenų struktūrų kūrimas pasirinktam ADT</h3>
	

		<p>Šis darbas yra išskirtinis, atliekamas savarankiškai, laboratorinių darbų grafike laikas neskiriamas. Jis skirtas įtvirtinti žinias, gautas pirmų trijų darbų metu. Galima naudotis interneto šaltiniais. <strong>Tačiau dėstytojui nustačius, kad studentas bando pristatyti kitų autorių atliktą darbą, yra skiriamos nuobaudos neigiamų vertinimų būdu.</strong></p>
<p><span style="text-decoration: underline;"><strong>Darbo tikslai:</strong></span></p>
<ol>
<li>Išmokti kurti naujas bendro naudojimo duomenų struktūrų klases pagal duotą (pasirinktą) ADT aprašą ;</li>
<li>Įtvirtinti projektavimo, programavimo, testavimo ir tyrimo procesų žinias;</li>
<li>Išmokti kurti programinės įrangos dokumentaciją;</li>
</ol>
<p><span style="text-decoration: underline;"><strong>Atsiskaitymas:</strong></span></p>
<ol>
<li>Pateikiamas visas programinis projektas ir ataskaitos elektroninė versija;</li>
<li>Pademonstruojamas sukurto projekto veikimas.</li>
<li>Atsakoma į klausimus apie programinės įrangos struktūrą, metodus, sprendimo pasirinkimo motyvus.</li>
</ol>
<p><span style="text-decoration: underline;"><strong>Darbo eiga</strong></span></p>
<p><em>Pradžioje </em>reikia apsispręsti dėl vieno iš galimų varijantų, kurį reikia pranešti laboratorinių darbų dėstytojui:</p>
<p style="padding-left: 30px;"><strong>1.</strong> Išnagrinėjamas pateiktas projektas, <span style="text-decoration: underline;">studentas savos duomenų struktūros nekuria; nei balai, nei nuobaudos neskiriami.</span> Rekomenduojama pasirinkti studentams, kuriems tolimesnėje veikloje pakanka programavimo pagal šablonus žinių.</p>
<p style="padding-left: 30px;"><strong>2.</strong> Pasirenkamas paskaitos metu nagrinėtas ar literatūroje aprašytas linijinės duomenų struktūros ADT (iš toliau pateikto sąrašo). Esant skirtingų realizacijų galimybei, konkretizuojamas realizacijos pobūdis.</p>
<p style="padding-left: 30px;"><strong>3.</strong> Galima pasirinkti duomenų struktūrų uždavinį iš kitų universitetų, pasinaudojant šiomis nuorodomis:</p>
<p style="padding-left: 90px;"><a href="http://introcs.cs.princeton.edu/java/assignments/">http://introcs.cs.princeton.edu/java/assignments/</a></p>
<p style="padding-left: 90px;"><a href="http://nifty.stanford.edu/">http://nifty.stanford.edu/</a></p>
<p style="padding-left: 30px;"><strong>4.</strong> Galima pateikti ir savo kuriamas programas, kuriose specializuotos duomenų struktūros išskirtos į atskirą klasę.</p>
<p style="padding-left: 30px;"><strong>5.</strong> C++ fanai gali sukurti vienos iš tiesinių struktūrų analogą C++ kalba ir atlikti Javos ir C++ programų efektyvumo analizę.</p>
<p style="padding-left: 30px;"><strong>6. </strong>Galima suderinti temą, kurioje nagrinėjamas abstraktus duomenų tipas, reikalingas sprendžiant diskrečiosios matematikos uždavinį (aibės - perinkimai, grafai, keliai, maršrutai), t.y. lygiagrečiai vykstančiame modulyje.</p>
<p><em>Reikia: </em>savarankiškai sukurti pasirinktą duomenų struktūros interfeisą ir ją realizuojančią klasę, jai sudaryti testo klases, atlikti efektyvumo tyrimą, pateikti demonstracinį panaudojimo varijantą.</p>
<p>Individualus varijantas būtinai suderinamas su laboratorinių darbų dėstytoju.</p>
<p>Galimų duomenų struktūrų sąrašas (tikslesni aprašai su paieška "data structures" + vardas):</p>
<p style="padding-left: 60px;">•    Stack - realizuota pavyzdyje<br />•    Queue- maksimalus balas 5<br />•    Deque- maksimalus balas 5<br />•    Doubly linked list - max 7<br />•    Priority queue<br />•    Bit array            •    Dynamic array      •    Circular buffer<br />•    Sparse array     •    Sparse matrix    <br />•    Set - įvairūs varijantai     •    Multiset<br />•    Unrolled linked list         •    VList         •    Skip list</p>
<p>Rūšiavimo metodų realizacija nurodoma atskirai ir priklauso nuo struktūros pobūdžio.<br />Pavyzdyje pateiktas steko klasės dviejų realizacijos variantų projektas.</p>
